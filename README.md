# Temi's Personal Site

> **Personal site where I share all my nusic interests!!**

**Live site:** [fairlyodd.blog](https://fairlyodd.blog/)

---

## What's Here

### Blog
A place for music reviews and articles. Posts are displayed in a card carousel with a grid toggle, fetched from my Wordpress.

### Stats
Music nerd tools powered by your [Last.fm](https://www.last.fm/) listening history and lyrics scraped from [Genius](https://genius.com/).

**Ear Words** — Enter your Last.fm username and a timeframe to see a word cloud built from the lyrics of your most-played songs. Word size reflects frequency.

**Lyrical Complexity** — Ranks your top tracks by lyrical complexity using NLP metrics: vocabulary diversity, average word length, and rare word ratio. Hover over any song for a full breakdown.

> **Note:** Genius is community-driven, so lyrics aren't always perfect. If a song shows weird data, it was probably matched to the wrong Genius page.

## Running Locally

**Frontend**
```bash
npm install
npm run dev        # http://localhost:3000
```

**Backend**
```bash
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
# create a .env with GENIUS_API_TOKEN=...
uvicorn main:app --reload
```

**Other scripts**
```bash
npm run test       # run tests with Vitest
npm run lint       # ESLint
npm run build      # production build
```

## Architecture Notes

- **File-based routing** — routes live in `src/routes/` and are auto-generated by TanStack Router. Each route owns its data loading via `useQuery` hooks.
- **Dependent queries** — the Stats pages chain two queries: first fetch top tracks from Last.fm, then pass those to the backend for lyric analysis. TanStack Query's `enabled` flag keeps these in sync.
- **NLP pipeline** — the FastAPI backend calls Last.fm for track metadata, scrapes Genius for lyrics, lemmatizes with NLTK, strips stop words, then scores each song.
